## 一、需求概述

### 1.1 项目背景

**摸鱼大乱斗**是一款基于现有摸鱼网站生态的轻量级 5v5 实时对战网页小游戏。旨在利用用户碎片化时间，提供快节奏、强社交的竞技娱乐体验，同时反哺主站流量与用户粘性。

### 1.2 核心价值主张

> **"10分钟一局，上班摸鱼也能玩的LOL大乱斗"**

- **低门槛**：无需下载，网页即开即玩
- **快节奏**：单线地图，无补刀发育，见面就是干
- **强社交**：内置语音/表情，促进职场友谊
- **轻惩罚**：掉线AI托管，随时可回归

### 2.2 间接竞品（移动端MOBA大乱斗）

表格





| 竞品                   | 关键数据/设计                                          | 核心洞察                                         |
| :--------------------- | :----------------------------------------------------- | :----------------------------------------------- |
| **王者荣耀-无限乱斗**  | 日活持续增长，平均在线时长显著提升；社交功能是粘性核心 | 随机BUFF是重玩价值关键；社交绑定比玩法深度更重要 |
| **LOL手游-极地大乱斗** | 单线地图，10-15分钟/局，无回城补给                     | 强调"持续团战"而非"运营发育"；死亡惩罚轻鼓励激进 |
| **CODM-空投区/重生**   | 快节奏5v5，短平快交战                                  | 小地图+二层设计提升策略性；随机复活点增加变数    |

### 2.3 竞品核心结论

**关键发现：**

1. **随机性是重玩价值核心**：王者荣耀无限乱斗通过随机BUFF提升用户留存 

   ，2. **社交绑定＞玩法深度**：用户更在意"和朋友一起玩"而非"游戏多复杂" 

   ；

2. **操作简化是趋势**：从PC端到移动端，从复杂操作到自动攻击+方向控制；

3. **轻量化必然牺牲部分策略深度**，但可通过"海克斯强化"类机制补偿。

**差异化机会：**

- 现有网页游戏多为"大球吃小球"或FPS，**无MOBA类大乱斗形态**
- 依托现有摸鱼网站用户池，**冷启动成本低于纯新作**
- 积分系统已就绪，**经济系统可直接复用**

## 三、核心玩法设计（MVP）

### 3.1 基础规则

表格





| 要素         | 设计                                             |
| :----------- | :----------------------------------------------- |
| **地图**     | 嚎哭深渊简化版：单线、无野区、草丛遮蔽、两侧基地 |
| **胜利条件** | 摧毁敌方基地 或 先达到30杀                       |
| **英雄**     | 全队随机分配（每局10选1，避免固化）              |
| **操作**     | 鼠标点击移动 + QWER施法（PC优先）                |
| **时长**     | 10-15分钟（死亡后8秒复活，无回城）               |

### 3.2 关键创新点

#### 3.2.1 "肾上腺素"节奏设计

- **0-3分钟**：试探性交火，技能交换
- **3-8分钟**：装备成型，团战爆发期（**核心爽点**）
- **8-12分钟**：决胜团战，一波推基地
- **12-15分钟**：垃圾时间/翻盘机会

#### 3.2.2 死亡与复活机制

- **死亡无惩罚**：仅丢失临时视野，金钱保留
- **8秒固定复活**：在基地满血满蓝，**鼓励激进不鼓励怂**
- **灵魂状态**：死亡后可透视战场3秒（给队友报点）

### 3.3 社交系统（MVP必做）

表格





| 功能         | 说明                                                      |
| :----------- | :-------------------------------------------------------- |
| **快速组队** | 自由加入蓝/红队，满10人自动开始倒计时                     |
| **房间系统** | 支持自定义房间+链接邀请（防陌生人）                       |
| **轻量互动** | 预设表情包（"nice!"、"我的锅"、"下班别走"）+ 一键点赞队友 |
| **战绩展示** | 本局KDA、伤害占比、承伤王等标签                           |

## 四、技术可行性评估

### 4.1 架构方案

plain

复制

```plain
用户端（React）
    ↓ WebSocket
网关层（Netty/WebFlux）← 心跳检测、房间管理、消息广播
    ↓ 业务服务（Java）
        - 对战服务：状态同步、伤害计算、复活逻辑
        - 匹配服务：简单ELO、房间管理
        - 用户服务：积分结算、战绩记录
    ↓ 数据层（Redis + MySQL）
        - Redis：实时房间状态、玩家位置缓存（TTL 5分钟）
        - MySQL：战绩持久化、排行榜
```

### 4.2 关键技术挑战与应对

表格





| 挑战             | 解决方案                                | 备注                                     |
| :--------------- | :-------------------------------------- | :--------------------------------------- |
| **实时同步延迟** | 客户端预测+服务端校验；位置同步100ms/次 | 技能释放走指令校验，非状态同步           |
| **断线重连**     | 5秒内重连保留状态；超5秒触发AI托管      | AI逻辑：跟随最近队友，优先攻击低血量敌人 |
| **并发性能**     | 房间隔离架构；单房间独立线程            | 目标100并发房间，单房间20人              |
| **防外挂**       | 服务端权威校验；关键数值加密传输        | 大乱斗模式相对MOBA，外挂影响较小         |

## 六、风险与应对

表格





| 风险点                              | 影响            | 应对策略                                                     |

## 七、亮点建议（差异化竞争）

### 7.1 "职场社交"场景深耕

- **午休局文化**：12:00-13:30高峰期匹配加速
- **部门对战**：支持自定义房间名"XX部门内战"
- **摸鱼伪装模式**：一键切换浏览器标签页为"Excel样式"（仅视觉层）

### 7.2 数据驱动的爽点设计

- **个人数据看板**：本周伤害总量、最常玩英雄、胜率曲线
- **团队数据对比**：部门胜率排行榜（激发集体荣誉感）

### 7.3 低门槛高上限策略

- **新手**：智能施法辅助（指示器显示技能范围）
- **进阶**：取消辅助，手动预判，解锁"精准"成就徽章
