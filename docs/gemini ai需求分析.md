# 《网页端极地大乱斗竞技模式需求分析文档 (PRD)》

## 1. 导言

### 1.1 文档目的

本需求分析文档旨在明确在“摸鱼网站”中嵌入一款基于5v5极地大乱斗模式的Web游戏的各项功能、技术指标及用户体验要求。本文档将作为开发团队进行技术架构设计、数值平衡调整及UI/UX开发的指导性基准。

### 1.2 业务背景

用户在工作间隙存在极强的社交与娱乐需求，传统的网页小游戏往往缺乏互动性和竞技深度。通过引入高人气的5v5单线对战模式及海克斯增强系统，旨在打造一个具有竞技吸引力的社交入口，促进网友情谊并提升网站活跃度。

## 2. 游戏模式与核心机制

### 2.1 极地大乱斗 (ARAM) 核心规则

- **地图规格**：采用“嚎哭深渊”风格的单线地图，双方基地各设有一座水晶枢纽、两座枢纽防御塔、两座外塔，以及分布在道路一侧的生命遗迹（Health Relics） 。
- **英雄分配**：开局全随机英雄，提供一次免费重随机机会。玩家可以从队友交换或放弃的“英雄池”中重新挑选。
- **数值修正体系**：
  - **基础补强**：英雄起始等级为3级，技能急速+70，被动每秒回复0.15%最大法力值 。
  - **平衡性平衡**：针对特定英雄实施百分比级别的伤害输出和承伤修正（如刺客类英雄增加韧性与伤害，远程消耗类英雄增加承伤倍率） 。
  - **距离减伤**：英雄受到距离1000码以外的敌方单位伤害时，根据距离获得15%至30%的伤害减免，防止远程Poke角色过度统治战场 。
- **战场机制**：
  - 禁止回城购买装备，玩家仅在阵亡后的复活时间内可在泉水商店操作。
  - 生命遗迹被拾取后，会产生一个类似于“救赎”的主动区域治疗效果，同时恢复拾取者的生命值与法力值 。

### 2.2 海克斯大乱斗强化符文系统（拓展功能）

- **抽取机制**：每局比赛在玩家等级达到3级、7级、11级、15级时，弹出强化符文选择界面，每位玩家从随机分配的三个符文中选择其一 。
- **符文分类**：
  - **白银级**：提供基础数值增强（如：提升10%移动速度，攻击附带微量真实伤害）。
  - **黄金级**：改变技能逻辑或交互（如：雪球变大且能穿过小兵，或者施放技能时对周围敌人造成震荡波） 。
  - **棱彩级**：提供足以扭转战局的质变能力（如：“尊我为王”在乘坐传送门后升级装备；“闪闪现现”获得额外的闪现技能） 。

## 3. 功能需求详细说明

### 3.1 核心战斗功能

- **角色控制**：支持鼠标点击移动（左键）与技能施放（键盘QWER/DF键），同时支持Web端流行的WASD移动模式以适应不同习惯。
- **技能系统**：每个角色包含1个被动技能、3个普通技能、1个终极技能。技能逻辑必须在服务器端计算以防止作弊 。
- **召唤师技能**：标配“闪现”与“雪球（标记）”。雪球击中敌人后，再次施放可突进至目标身边，这是乱斗模式开团的核心手段。
- **商店与装备**：精简版装备体系，包含神话/传说级核心装及基础属性装。支持自动购药与推荐出装功能，降低摸鱼玩家的操作复杂度。

### 3.2 匹配与社交功能

- **快速匹配**：基于简化的ELO分值进行撮合，支持单人或组队排队。
- **一键邀请**：生成专属链接，网友点击链接即可直接进入当前房间的等待列表。
- **局内互动**：提供快捷语轮盘（如“摸鱼中，求带飞”、“老板来了，稳住”）及表情包系统，促进网友间的即时社交。

### 3.3 摸鱼专属特色功能 (Mo-Fish Features)

- **智能老板键 (Panic Switch)**：
  - **功能描述**：按下特定快捷组合键，游戏画面立即消失，浏览器当前标签页显示为预设的办公内容。
  - **静音逻辑**：开启老板键后，全局音频自动静默。
- **UI 隐蔽模式 (Stealth UI)**：
  - **功能描述**：一键切换“专业图表”风格皮肤，将游戏主视觉转化为低对比度的类似工作软件的界面。
- **断线重连与 AI 接管**：
  - **功能描述**：当检测到玩家因关闭标签页或窗口失去焦点超过30秒时，系统自动指派AI接管角色。玩家返回页面后可无缝恢复控制。

## 4. 技术与性能要求

### 4.1 性能指标

- **渲染帧率**：在典型办公电脑（集成显卡）环境下，5v5团战帧率不得低于 45 FPS。
- **首屏加载**：游戏初始核心资源包体积控制在 5MB 以内，后续资产采用按需加载模式，加载时间不超过 5 秒。
- **网络延迟**：国内主流网络环境下，操作延迟（RTT）应控制在 50ms 以内，通过 WebRTC 协议实现低延迟数据交换 。

### 4.2 兼容性要求

- **浏览器支持**：兼容 Chrome 90+, Firefox 88+, Safari 14+ 及主流 Chromium 内核浏览器。
- **硬件适配**：支持从低功耗超极本到高性能台式机的平滑缩放，自动根据硬件性能调整粒子特效密度。

## 5. 资源与资产需求

### 5.1 视觉资产 (Visual Assets)

- **角色设计**：首批上线 10-15 名英雄，采用扁平化或低面数（Low-Poly）风格，兼顾美观与加载效率。
- **环境地图**：原创“嚎哭深渊”风格单线地图，避免使用受版权保护的特定视觉元素。

### 5.2 音频资产 (Audio Assets)

- **战斗音效**：打击感强的原创音效，包含技能命中、金币获得、防御塔损毁等反馈。
- **系统配音**：幽默、轻松的解说配音，贴合“摸鱼”主题。

## 6. 成本与进度估算建议

根据行业同类项目调研，本项目的初步落地分为三个阶段。

| **阶段**                 | **周期** | **核心交付物**                                      | **预计成本占比** |
| ------------------------ | -------- | --------------------------------------------------- | ---------------- |
| 第一阶段：原型开发 (MVP) | 2-3 个月 | 基础5v5同步架构, 单英雄对战Demo, 老板键功能         | 30%              |
| 第二阶段：系统完善       | 3-4 个月 | 完整的英雄池(10+), 商店装备系统, 初版海克斯强化系统 | 40%              |
| 第三阶段：打磨与上线     | 1-2 个月 | 性能优化, 社交互动功能集成, 匹配系统上线            | 30%              |



本方案通过将极地大乱斗的爽感与网页端的轻量化便捷性相结合，配合深度的“摸鱼”定制化设计，能够有效验证想法的可靠性。该项目不仅具备极高的用户吸引力，更由于其在Web端填补了高品质MOBA的空白，具有显著的长期运营价值。
