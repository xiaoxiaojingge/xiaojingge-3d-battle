一、项目概述与背景
1.1 项目目标
本项目旨在使用 Three.js 技术栈，在浏览器环境中构建一个高度还原《英雄联盟》极地大乱斗
（ARAM）游戏场景的3D可交互Demo。核心目标包括：
场景还原：精确复刻嚎哭深渊（Howling Abyss）地图的地形、建筑、光照与氛围
可交互性：支持玩家操控英雄角色在地图中移动、释放技能、攻击敌方单位
机制还原：实现ARAM核心游戏规则，包括小兵生成、防御塔攻击、生命圣物等
80%体验还原：在视觉效果、操作手感、游戏节奏三个维度达到原版80%的沉浸感
1.2 核心定位
产品定位
本Demo不是一个完整的商业游戏，而是一个技术展示型3D交互Demo，用于：
展示WebGL/Three.js在浏览器3D游戏开发中的技术能力
作为3D场景构建、游戏机制实现的学习参考项目
验证浏览器端MOBA类游戏核心体验的技术可行性

二、竞品分析：原版LOL极地大乱斗
2.1 地图概述
嚎哭深渊（Howling Abyss）是《英雄联盟》中专为ARAM模式设计的单线地图，位于弗雷尔卓
德（Freljord）的冰原深处。地图以一座横跨深渊的狭窄桥梁为核心战场，两侧是冰雪覆盖的悬
崖。该地图自2013年V3.6版本取代试炼之地（Proving Grounds）后，一直是ARAM模式的官方
地图。
地图设计哲学：嚎哭深渊的设计核心是"强制团战"——单线结构消除了多线运营的策略空
间，将游戏焦点完全集中在5v5团队战斗上，平均游戏时长控制在15-20分钟。

十、结论与建议
项目价值评估
本项目具有较高的技术展示价值和学习价值。通过复刻英雄联盟极地大乱斗这一经典场景，可以
全面展示Three.js在以下方面的能力：
大规模3D场景构建与优化
俯视角摄像机控制系统
游戏AI（小兵、防御塔）实现
技能系统与碰撞检测
Web端3D游戏性能调优
核心建议
1. 坚持Demo定位，不贪求全功能
核心目标是一个"可玩的技术展示"，而非完整游戏。优先保证场景氛围和操控手感，而非复
杂的英雄技能系统。
2. 采用低多边形美术风格
低多边形（Low-Poly）风格既能减少建模工作量，又具有独特的美学价值，同时大幅降低渲
染压力。冰雪场景非常适合这种风格。
3. 优先实现核心体验闭环
最小可玩版本（MVP）应包含：移动 → 普攻 → 击杀小兵 → 摧毁防御塔。在这个核心循环
跑通后再扩展技能系统和视觉效果。
4. 建立性能监控机制
从项目第一天就集成Stats.js性能面板，每增加一个功能都验证FPS影响。性能优化是渐进的
过程，不能留到最后。
