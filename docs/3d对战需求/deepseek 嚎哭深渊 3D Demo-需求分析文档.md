一、项目概述
1.1 项目背景
本项目的目标是用 Three.js 开发一款英雄联盟“极地大乱斗”模式（嚎哭深渊地图）的 3D Web Demo，力求在浏览器环境中还原该模式的核心视觉与操作体验。用户可以通过浏览器直接体验在一个狭窄冰雪桥梁上展开的 5v5 激烈团战，感受 MOBA 核心玩法的 3D 化呈现。

1.2 项目目标
实现一个单局可玩、体验流畅的 3D 场景游戏 Demo，完成度达到原游戏体验的 80% 左右，覆盖以下核心维度：

场景还原：嚎哭深渊地图的视觉风格、建筑布局、草丛分布、氛围特效

核心机制：英雄控制、战斗系统（普攻/技能）、兵线机制、防御塔攻击、胜利条件

操作手感：斜 45 度俯视固定视角（类似原版 RTS 风格）、流畅的移动与技能施放

性能表现：Three.js 渲染流畅，PC 浏览器稳定运行

二、竞品调研
2.1 原版《英雄联盟》嚎哭深渊地图
维度	详细说明
地图背景	嚎哭深渊是一个无底裂隙，位于弗雷尔卓德最为寒冷、最为残酷的部分。传说在很久以前，一场宏大的战役就发生在横跨这道天堑的一座狭窄桥梁上。
地图结构	只有一条道路，位置和形状约等于召唤师峡谷里的中路，没有任何岔路，只有双方水晶附近比较开阔（类似于高地），以及墙边有少许的凹凸部分。
建筑配置	双方各有：位于道路中央的两座外塔、一个兵营（小水晶/抑制剂）、两颗门牙塔和一个水晶枢纽（主基地）。除了主水晶，所有建筑物都不会自动回复生命值。
草丛布局	全图共有5片草丛区域：两片靠近地图中央两侧，双方外塔与高地塔之间各有一片。12.23版本后新增了桥中底部的草丛。
特殊机制	泉水无治疗效果，一旦离开基地后只有死亡才能再次在商店购买道具，道路上会定期刷新治疗圣坛（血包）。
冰霜飞门	12.23版本新增，从水晶枢纽传送至外塔，当己方首个防御塔告破后传送门会连接到召唤水晶塔。
防御塔碎石	外侧防御塔被摧毁后会留下一排碎石，这些碎石将被视为新的可破坏地形。
魄罗	地图上有可爱的魄罗小动物，可以喂食魄罗饼干让它们变大。
视角风格	斜 45 度俯视固定视角（等距视角，Isometric-like），相机角度固定，不能自由旋转，但可滚动缩放和边缘移动。
核心玩法分析：嚎哭深渊的设计哲学是“鼓励打团”。只有一条兵线，没有打野干扰，没有分推战术，所有经济来源集中于兵线击杀与助攻奖励。治疗包刷新周期约60秒，草丛提供绝对视野压制。2026年2月版本后治疗包回复量调整为 8% + 已损失生命值 4%，强化了资源争夺策略。

竞品优势总结：

✅ 地图结构极其简单（单一兵线），适合 3D 重构

✅ 游戏节奏快（20-30分钟一局），适合 Demo 展示

✅ 核心机制清晰（推塔→胜利），易于实现 MVP

✅ 无回城机制简化了状态管理逻辑

✅ 玩家认知度高，易于获得反馈

// 斜45度固定相机，看向地图中心或跟随英雄
const cameraDistance = 1000;
const angleX = 45 * Math.PI / 180;  // 水平角45度
const angleY = 35 * Math.PI / 180;  // 俯角35度

let targetX = hero ? hero.position.x : 0;
let targetZ = hero ? hero.position.z : 0;

let offsetX = cameraDistance * Math.cos(angleX) * Math.cos(angleY);
let offsetZ = cameraDistance * Math.sin(angleX) * Math.cos(angleY);
let offsetY = cameraDistance * Math.sin(angleY);

camera.position.set(targetX + offsetX, targetY + offsetY, targetZ + offsetZ);
camera.lookAt(targetX, targetY, targetZ);
4.7.3 视角控制逻辑总结

八、总结与建议
8.1 核心结论
项目可行性高：Three.js 生态成熟，有足够的开源资源支撑开发，技术风险可控。

市场需求存在：目前缺少完整的 3D MOBA Web Demo，本项目可填补这一空白。

核心参数清晰：嚎哭深渊地图结构简单（一条兵线），建筑和机制明确，易于实现。

视角方案明确：采用斜45度俯视固定相机，还原原版游戏最经典的视角体验，避免了自由环绕带来的眩晕和复杂度。

可实现 80% 还原度：在模型资源、碰撞检测和技能系统等方面进行适当简化后，完全可以在合理开发周期内达到 80% 的体验还原。

8.2 优先建议
聚焦 MVP 先行：第一版只实现 1 条兵线、2 个防御塔、1 个英雄、普攻 + 1 个技能，验证核心玩法后再迭代。

模型资源优先使用开源：利用现成的 LOL 3D 模型 Gallery 项目，避免从零建模。

视角选择斜45度固定相机：这是原版最经典、最易实现的方案，配合边缘平移和滚轮缩放即可获得极佳体验。

操作采用鼠标右键移动：与英雄联盟原版操作习惯一致，用户上手成本低。

8.3 最终判断
该项目值得投入，且有较大概率成功产出高质量的 3D 游戏 Demo。

✅ 技术难度：中等，Three.js 学习曲线平缓

✅ 开发周期：2-3 个月可获得可玩版本

✅ 学习价值：完整的 3D 游戏开发流程，可积累大量实战经验

✅ 展示价值：可作为作品集的核心项目

✅ 扩展性：后续可增加英雄、技能、联网对战等功能
