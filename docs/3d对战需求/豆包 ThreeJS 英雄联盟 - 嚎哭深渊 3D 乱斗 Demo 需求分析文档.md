一、项目概述
1.1 项目背景
英雄联盟「极地大乱斗 - 嚎哭深渊」是玩家热度最高的轻量化团战模式，拥有固定线性地图、对称场景、锁定俯视角、草丛视野博弈、极简战斗规则等标志性特征。目前全网缺少高还原度、基于 ThreeJS 的 Web 端 3D 嚎哭深渊复刻 Demo，多数同类 Demo 仅实现简单地形行走，缺失草丛视野、固定视角、建筑碰撞、场景细节核心能力。
1.2 项目目标
基于 ThreeJS 开发单机版 3D 场景 Demo，完整复刻嚎哭深渊地图结构、建筑分布、草丛位置、大乱斗专属视角、键鼠操作规则、场景氛围，整体达到原版游戏 80% 体验还原度；仅弱化战斗逻辑（兵线、技能伤害、多人联机），聚焦场景 + 视角 + 移动 + 视野核心体验。
1.3 文档目的
完成竞品调研、项目可行性验证、原版场景全参数拆解、核心需求定义、亮点拓展设计，为后续 ThreeJS 代码开发、模型搭建、逻辑编写提供标准化依据。

2.2 项目可行性分析
技术可行性【完全可行】
ThreeJS 原生支持 3D 地形搭建、模型导入、碰撞检测、相机锁定、图层视野遮罩、粒子特效、键鼠输入监听；大乱斗场景为低多边形冰雪风格，地形结构简单（单一线性地图、无复杂高低差），开发难度中等，无技术壁垒。
资源可行性【成本可控】
嚎哭深渊为对称冰雪场景，可使用免费低模、程序化地形搭建、极简贴图完成复刻；个人 Demo 非商用，可通过结构复刻 + 原创低模规避版权问题。
开发成本【低门槛】
无需服务端、无需联机、无需战斗数值逻辑，纯前端单机实现；模块化拆分（场景、相机、碰撞、操作、视野），可分阶段迭代开发。
2.3 项目价值 & 落地结论
学习价值：整合 ThreeJS 光照、碰撞、遮罩、动画、相机约束等核心能力，是 Web3D 优质实战项目；
差异化价值：填补全网高还原度嚎哭深渊 ThreeJS Demo 空白，可作为技术展示作品；
拓展价值：后续可无缝迭代 AI 人机、技能释放、多人联机、特效系统；
✅ 最终结论：项目想法高度可靠、技术完全落地、开发成本低、拓展性极强，完全值得开发，80% 原版体验还原目标可精准落地。
三、原版嚎哭深渊 全维度核心参数（开发核心依据）
统一标准：采用 LOL 官方游戏单位，1 游戏单位 = ThreeJS 1 米，直接等量换算开发，保证尺寸 1:1 还原。
3.1 地图基础规格
地图形态：水平线性对称单路地图，整体矩形闭环战斗区域，无高低落差、无野区、无分支路线；
整体尺寸
地图总长度（蓝方基地→红方基地中轴线）：10200 单位
地图总宽度（左右两侧悬崖边界）：2800 单位
核心战斗主干道宽度：650~720 单位
两侧不可通行悬崖区域：左右各 1050 单位，永久碰撞封锁
地形风格：弗雷尔卓德暗黑冰雪风，冻土 + 破碎石台 + 断桥 + 冰封残骸，全局冷色调地表材质。
3.2 全局建筑分布（位置 + 功能 + 碰撞属性）
地图严格红蓝对称，从蓝方→红方依次排布：
双方出生基地（端点建筑）
位置：地图最左端（蓝）、最右端（红）；
模型组成：冰雪复活平台、弗雷尔卓德石雕、冰封石柱、破碎城墙；
碰撞规则：平台内部自由通行，外围设置隐形空气墙，限制开局脱离出生区；
专属规则：大乱斗基地无持续回血，仅复活满血。
防御塔（双方各 1 座，大乱斗唯一防御建筑）
坐标：蓝塔距蓝方基地 1850 单位，红塔距红方基地 1850 单位，中轴线对称；
模型：冰霜石质高塔、魔法水晶塔顶、石质围墙；
碰撞：塔体实心碰撞（不可穿越），攻击范围球形碰撞体（半径 800 单位）。
中央中立废墟建筑
位置：地图正中央（中轴线 5100 单位）断桥区域；
模型：断裂石柱、冰封骸骨、破碎祭坛、残骸战船；
碰撞：部分残骸为小型阻挡碰撞，核心主干道无阻挡。
边界山体围墙
贯穿地图左右全程的冰雪悬崖山体，永久实体碰撞，强制限制单路战斗范围。
3.3 草丛详细分布（大乱斗共 6 组对称草丛，视野核心机制）
草丛核心规则：角色进入草丛→敌方视野完全屏蔽，友方视野正常，草丛可自由穿行、无实体碰撞。

蓝方右草 C2	蓝防御塔前方 450 单位 主干道右侧	320×180 单位

中央右草 Z2	地图中点断桥右侧	380×200 单位

补充参数：草丛植被高度 0.8 单位，半透明灌木模型，仅做视野遮罩渲染，不阻挡移动。
3.4 大乱斗专属视角设定（核心差异化规则）
相机类型：固定锁定斜 45° 俯视角，禁止自由旋转、禁止大幅度俯仰；
相机硬性参数（1:1 还原）
俯视角度：垂直地面夹角 38°
水平朝向：固定 45° 斜向（西北→东南全局统一）
基础视距：默认镜头高度 1280 单位
缩放限制：鼠标滚轮仅支持小幅缩放，区间 850~1500 单位；
视角锁定规则
默认开启「Y 键英雄跟随锁定」；
禁止镜头左右旋转、禁止拖拽移动地图；
空格键快速复位至自身英雄视角；
视野图层规则
全地图无战争黑雾，地形永久全亮；
仅草丛独立图层遮罩，实现视野隔断效果。
3.5 原版操作按键全规格（键鼠适配）
移动操作
鼠标右键：地面点击移动、长按持续行走；
视角快捷键
Y：锁定 / 解锁英雄跟随视角；
鼠标滚轮：镜头拉近 / 拉远（限制区间）；
空格：快速切回自身视角；
功能按键
Q/W/E/R：技能按键（预留接口）；
D/F：召唤师技能（预留接口）；
ESC：取消操作、呼出菜单；
移动碰撞约束
不可穿越墙体、防御塔、山体；
草丛、中央断桥无移动阻挡；
地图两端隐形空气墙，防止出界。
3.6 场景氛围细节参数
光照：全局冷色调漫反射，阴天无强光，柔和阴影，冰雪蓝白色温；
粒子特效：持续全局微降雪、寒风动态粒子、冰封微光特效；
材质：地面结冰纹理、岩石冰霜材质、建筑复古石质贴图；
环境音：预留风声、深渊背景音、环境底噪音频接口。

六、非功能性需求
性能需求：Web 端浏览器稳定 60FPS，模型轻量化，资源加载时长＜3 秒；
兼容性：适配 Chrome/Edge 主流现代浏览器，支持 PC 端键鼠操作；
可拓展性：代码模块化拆分（场景模块 / 相机模块 / 碰撞模块 / 视野模块），便于后续迭代；
可维护性：参数配置化（地图尺寸、草丛坐标、相机参数统一配置文件），修改无需改动核心逻辑。
七、项目风险 & 解决方案
风险 1：草丛视野遮罩实现困难
解决方案：采用 ThreeJS图层分层渲染 + 遮罩材质，单独划分草丛视野图层，实现敌方单位隐藏效果；
风险 2：3D 碰撞检测繁琐
解决方案：简化碰撞体，建筑 / 山体使用盒形碰撞体，地形使用 Octree 轻量化碰撞检测；
风险 3：官方模型版权问题
解决方案：不使用 LOL 官方原模，通过结构复刻 + 开源低模 + 原创建模还原外观，规避版权风险。
