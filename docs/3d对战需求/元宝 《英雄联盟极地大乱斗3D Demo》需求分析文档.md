# 《英雄联盟极地大乱斗3D Demo》需求分析文档

## 一、项目概述

### 1.1 项目背景
本项目旨在使用Three.js技术栈开发一个英雄联盟极地大乱斗场景的3D交互式Demo，目标实现80%的原游戏体验还原度。极地大乱斗作为英雄联盟最受欢迎的游戏模式之一，具有单线作战、快节奏、随机英雄等特色，适合作为技术演示场景。

### 1.2 项目目标
- 技术验证：探索Three.js在复杂游戏场景中的应用能力
- 体验还原：实现嚎哭深渊地图的3D可视化与基础游戏交互
- 教育价值：为WebGL/Three.js学习者提供实战参考案例
- 创新尝试：在浏览器端实现MOBA类游戏的轻量化体验

## 二、竞品调研与技术可行性分析

### 2.1 现有竞品分析
根据搜索结果，已有部分Three.js实现的LOL相关项目：
- **LOL-Web项目**：实现了基础的地图渲染和角色控制
- **hzwjxc/lol_threejs**：开源Three.js LOL Demo项目
- **联盟宇宙地图**：展示了英雄联盟世界观地图的3D可视化

**竞品不足**：
1. 功能完整性较低，多为技术演示
2. 游戏机制还原度有限
3. 缺乏极地大乱斗特色系统
4. 交互体验较为基础

### 2.2 技术可行性评估
**优势**：
1. Three.js成熟的3D渲染能力支持复杂场景
2. WebGL在现代浏览器中普及率高
3. 已有相关开源项目可参考
4. 英雄联盟API和资源相对丰富

**挑战**：
1. 性能优化：确保在浏览器中流畅运行
2. 资源管理：模型、贴图、音效等资源加载
3. 网络同步：如需多人联机，需考虑网络架构
4. 版权问题：需注意资源使用的合法性

## 三、核心需求分析

### 3.1 地图场景需求
#### 3.1.1 地图尺寸与布局
- **地图长度**：约3000码（2700米）
- **地图宽度**：单线桥式结构，宽度需参考原游戏比例
- **对称设计**：地图呈轴对称布局

#### 3.1.2 建筑与地形元素
| 元素类型 | 蓝色方（左） | 红色方（右） | 中央区域 |
|---------|------------|------------|---------|
| 防御塔 | 2座前防御塔 | 2座前防御塔 | 无 |
| 水晶防御塔 | 2座 | 2座 | 无 |
| 兵营 | 1个 | 1个 | 无 |
| 水晶枢纽 | 1个 | 1个 | 无 |
| 草丛 | 外塔与高地塔间草丛 | 外塔与高地塔间草丛 | 桥中底部新增草丛 |
| 治疗点（生命圣坛） | 4个固定位置 | 4个固定位置 | 随机刷新医疗包 |

#### 3.1.3 特殊地形机制
1. **防御塔废墟**：防御塔被摧毁后留下碎石地形
2. **海克斯传送门**：外塔被毁后连接己方水晶防御塔
3. **魄罗系统**：地图中的可爱生物，可投食互动

### 3.2 视角系统需求
#### 3.2.1 摄像机设定
- **视角类型**：固定斜45度俯视角（类似原游戏）
- **视野范围**：拉高的摄像机视野，平衡长距离技能
- **镜头控制**：支持拖拽移动、缩放、视角旋转
- **英雄跟随**：空格键锁定/解锁英雄视角

#### 3.2.2 视野机制
- **战争迷雾**：实现基础的视野系统
- **草丛隐身**：进入草丛后敌方不可见
- **视野障碍**：利用防御塔废墟等地形阻挡视野

### 3.3 操作控制系统
#### 3.3.1 双模式操作支持
| 操作模式 | 移动控制 | 技能释放 | 优势 |
|---------|---------|---------|------|
| **传统模式** | 鼠标右键点击移动 | QWER+鼠标指向 | 老玩家习惯 |
| **WASD模式** | WASD键盘移动 | 鼠标右键+Shift+E+R | 精准走位，新玩家友好 |

#### 3.3.2 核心按键映射
- **移动**：鼠标右键（传统） / WASD（新模式）
- **技能**：Q、W、E、R
- **召唤师技能**：D、F
- **物品**：1-6数字键
- **视角锁定**：空格键
- **聊天**：Enter（团队）/ Shift+Enter（全体）

### 3.4 游戏机制需求
#### 3.4.1 大乱斗特色系统
1. **初始状态**：
   - 初始等级：3级，3个技能点
   - 初始金钱：1375金币
   - 泉水无治疗效果，回蓝速度加快

2. **购物限制**：
   - 离开基地后，只有死亡才能再次购物
   - 部分道具不允许购买

3. **经验与金币**：
   - 持续获得经验
   - 被动金币增长

4. **治疗系统**：
   - 固定位置生命圣坛
   - 随机刷新医疗包

#### 3.4.2 英雄系统
- **随机选择**：系统随机分配英雄
- **重随机制**：消耗重随点数更换英雄
- **技能系统**：实现基础技能逻辑
- **属性平衡**：近战英雄获得10魔抗加成

### 3.5 视觉与音效需求
#### 3.5.1 视觉风格
- **主题**：弗雷尔卓德冰雪风格
- **色调**：冷色调为主，蓝白冰雪场景
- **特效**：技能特效、命中反馈、伤害数字
- **UI设计**：还原LOL经典UI布局

#### 3.5.2 音效系统
- **环境音效**：寒风呼啸、深渊回响
- **技能音效**：基础技能音效
- **交互反馈**：点击、移动、攻击音效
- **语音播报**：First Blood、Double Kill等

## 四、技术架构设计

### 4.1 技术栈选择
| 层级 | 技术选型 | 说明 |
|------|---------|------|
| **3D引擎** | Three.js r79+ | 主流WebGL库，成熟稳定 |
| **物理引擎** | Cannon.js / Ammo.js | 可选，基础碰撞检测 |
| **网络通信** | Socket.io | 如需多人联机 |
| **UI框架** | HTML5 + CSS3 | 传统Web技术 |
| **构建工具** | Webpack / Vite | 模块打包与开发环境 |
| **资源管理** | Three.js Loader | 模型、纹理、音效加载 |

### 4.2 核心模块设计
1. **场景管理模块**：地图加载、对象管理、渲染优化
2. **角色控制模块**：移动、技能、状态管理
3. **视角控制模块**：摄像机、视野、镜头切换
4. **游戏逻辑模块**：伤害计算、状态机、规则引擎
5. **网络同步模块**（可选）：状态同步、延迟补偿
6. **资源管理模块**：加载、缓存、释放策略

### 4.3 性能优化策略
1. **LOD系统**：根据距离调整模型细节
2. **视锥体剔除**：只渲染可见物体
3. **实例化渲染**：批量渲染相同模型
4. **纹理压缩**：使用KTX2等压缩格式
5. **资源预加载**：游戏开始前加载关键资源

## 五、创新亮点与差异化

### 5.1 技术亮点
1. **双操作模式无缝切换**：同时支持传统点击和WASD操作
2. **动态地形系统**：实现防御塔废墟的地形变化
3. **视野系统还原**：精确模拟草丛隐身和战争迷雾
4. **Web端性能优化**：在浏览器中实现流畅的3D MOBA体验

### 5.2 体验亮点
1. **快速体验**：无需下载客户端，浏览器直接游玩

3. **自定义内容**：支持地图编辑、英雄数据调整
4. **社交分享**：一键生成游戏录像分享链接

### 5.3 扩展可能性
1. **地图编辑器**：可视化编辑嚎哭深渊地形
2. **英雄编辑器**：自定义英雄技能和属性
3. **观战系统**：第三人称观战视角
4. **回放系统**：游戏录像保存与回放

## 八、结论与建议

### 8.1 项目可行性结论
**高度可行**，理由如下：
1. 技术栈成熟：Three.js完全有能力实现此类3D场景
2. 参考资源丰富：有开源项目和详细文档参考
3. 目标明确：80%还原度目标实际可达
4. 市场需求：Web端3D游戏体验有明确需求

### 8.2 实施建议
1. **采用MVP策略**：先实现核心玩法，再逐步完善
2. **注重性能优化**：Web端性能是关键成功因素
3. **合法合规**：注意版权问题，使用原创或合规资源
4. **社区参与**：考虑开源，吸引开发者共同完善

### 8.3 成功关键指标
1. **性能指标**：主流设备60FPS稳定运行
2. **还原度指标**：核心玩法机制80%以上还原
3. **用户体验**：操作流畅，学习成本低
4. **扩展性**：代码结构清晰，易于功能扩展
