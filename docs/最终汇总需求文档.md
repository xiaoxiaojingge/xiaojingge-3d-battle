# 《摸鱼大乱斗》需求分析文档

## 1. 需求背景

“摸鱼网站”是一个面向职场人群的休闲社交平台，用户在工作间隙存在强烈的轻娱乐和社交需求。现有网页小游戏普遍缺乏互动深度与竞技性，导致用户停留时间短、粘性低。

《英雄联盟》中的“极地大乱斗”（ARAM）模式凭借单线快节奏、随机英雄、持续团战的特点，深受玩家喜爱；后续的“海克斯大乱斗”更通过随机增益增加了重复可玩性。

因此，我们计划在摸鱼网站中嵌入一款**5v5实时对战网页游戏**（暂定名：**摸鱼大乱斗**），采用类似嚎哭深渊的单线地图，降低操作门槛（无补刀、无回城），强调“10分钟一局、见面就团”的爽感，并深度结合摸鱼场景的特色功能（如老板键、伪装界面），从而促进网友友谊、提升网站活跃度与用户粘性。

## 2. 目标人群

| 维度         | 描述                                                         |
| :----------- | :----------------------------------------------------------- |
| **核心用户** | 摸鱼网站现有注册用户，年龄 22~35 岁，职场白领、程序员、运营等 |
| **场景**     | 工作午休、会议间隙、下班前等碎片化时间，单局期望时长 ≤15 分钟 |
| **行为特征** | 喜欢轻竞技、强社交、低学习成本；对“上班玩游戏”有隐蔽性需求   |
| **设备环境** | 主要为公司配发的办公电脑（集成显卡、Chrome/Edge 浏览器），少量移动端 |

## 3. MVP 最小可行功能点

> **原则**：只做最核心的对战闭环，确保用户能在 3 个月内玩到“可联机、能打架、有输赢”的完整游戏。以下功能必须全部实现，且开发难度可控，可一口气做完。

### 3.1 核心对战

- **5v5 实时对战**：单线地图（无野区、无防御塔以外的中立建筑），10 名玩家同时在线。
- **英雄随机分配**：每局开始前为每位玩家随机分配一个英雄（从预设的 10~15 个英雄池中抽取），无重复英雄（红蓝方可出现相同英雄）。
- **基础战斗逻辑**：
  - 鼠标点击移动 + QWER/DF 施放技能（仅支持 PC 键鼠）。
  - 每个英雄包含 1 个被动、3 个小技能、1 个终极技能。技能逻辑由服务器计算（防作弊）。
  - 普攻自动攻击最近的敌方单位（简化操作，类似“自动攻击”选项）。
- **召唤师技能**：标配“闪现”（短距离位移）和“雪球”（标记命中后可突进），这是大乱斗的核心开团手段。
- **死亡与复活**：
  - 死亡后 8 秒固定时间在基地复活，满血满蓝。
  - 死亡无金钱惩罚，不掉落任何资源。
- **胜利条件**（二选一触发）：
  - 摧毁敌方基地水晶。
  - 先达到 **30 次团队击杀**（防止僵局，加快节奏）。

### 3.2 组队与匹配

- **快速房间模式**：
  - 用户进入游戏大厅后，可自由选择加入“蓝队”或“红队”。
  - 当两队均满 5 人时，自动开始 5 秒倒计时并进入对局。
- **邀请链接**：房主可生成一个专属链接，网友点击后直接加入当前房间（无需登录摸鱼网站？需跳转并校验身份）。
- **AI 补位**：若排队等待超过 45 秒，剩余空位由 AI 机器人补齐，保证对局快速开始（AI 难度简单，仅作为填充）。

### 3.3 局内社交（轻量级）

- **快捷表情/语轮盘**：预设 6 个互动选项（如“摸鱼中，求带飞”、“我的锅”、“NICE！”、“老板来了”），点击后屏幕飘文字。
- **战绩面板**：实时显示本局每个玩家的 KDA（击杀/死亡/助攻）、伤害占比，游戏结束后展示全场最佳（伤害最高或击杀最多）。

### 3.4 数据与激励（对接现有系统）

- **积分结算**：
  - 胜利方每位玩家获得 +20 网站积分，失败方 +5 积分。
  - 连胜额外奖励（+5 积分/每场连胜，上限 3 场）。
- **战绩记录**：记录玩家的总场次、胜场、总击杀、MVP 次数，提供个人数据看板（只读）。

### 3.5 技术保障（简化版）

- **实时同步**：采用 WebSocket + 状态同步（服务器定时广播位置，100ms/次），客户端做简单插值。
- **断线重连与 AI 托管**：
  - 玩家断开连接后，保留其英雄 30 秒；若 30 秒内未重连，则 AI 接管该英雄（AI 逻辑：跟随最近的队友，攻击最近的敌人）。
  - 玩家重新进入房间后可立刻恢复控制。
- **性能要求**：在集成显卡办公电脑上，5v5 团战场面帧率 ≥ 30 FPS；核心资源包首屏加载 ≤ 3MB，加载时间 ≤ 5 秒。

## 4. 后续扩展点

> 按**功能重要性**（高/中/低）和**实现复杂度**（高/中/低）排序，建议在 MVP 上线验证数据后再迭代。

| 扩展功能                   | 描述                                                         | 重要性                          | 复杂度 | 建议阶段 |
| :------------------------- | :----------------------------------------------------------- | :------------------------------ | :----- | :------- |
| **海克斯强化系统**         | 每局游戏在等级 3/7/11/15 级时，玩家从三个随机符文（白银/黄金/棱彩）中选择一个，获得特殊能力（如技能冷却-20%、闪现额外充能等）。这是提升重复可玩性的关键。 | 高                              | 中     | 第二阶段 |
| **装备商店系统**           | 简化版装备（6 个格子），阵亡期间可用金币购买属性装（只保留攻击力/法强/生命值等基础属性，无主动装备）。 | 中                              | 中     | 第二阶段 |
| **英雄池扩充**             | 从 MVP 的 10~15 个英雄逐步增加至 30+，每个英雄有独特技能机制。 | 高                              | 中     | 持续进行 |
| **老板键（Panic Switch）** | 按特定组合键（如 `Ctrl+Shift+B`）一键隐藏游戏画面，浏览器标签页显示预设办公内容（如 Excel 图表），并自动静音。 | 中（摸鱼特色）                  | 低     | 第二阶段 |
| **UI 隐蔽模式**            | 一键切换低对比度“专业图表”皮肤，使游戏界面看起来像工作软件。 | 中                              | 低     | 第二阶段 |
| **排位/天梯匹配**          | 基于 ELO 分值的真匹配系统，取代简单的随机房间，增加竞技深度。 | 低（MVP 的邀请+房间已满足社交） | 高     | 第三阶段 |
| **观战系统**               | 死亡后可观看队友视角，或好友可旁观对局。                     | 中                              | 中     | 第三阶段 |
| **回放/精彩时刻**          | 自动录制击杀瞬间，生成 GIF 或视频分享。                      | 低                              | 高     | 第三阶段 |

## 6. 需求功能核心清单（Task 列表）

> 以下为 **MVP 必须完成** 的任务清单，采用 GitHub Task List 格式，方便追踪进度。

- **服务端基础架构**
  - 搭建 WebSocket 网关（支持房间管理、消息广播）
  - 实现房间数据结构（蓝队/红队、玩家列表、房主权限）
  - 实现简单匹配逻辑（两队满5人自动开局）
- **客户端基础**
  - 游戏大厅界面（显示房间号、队伍列表、邀请链接生成按钮）
  - 游戏主界面（单线地图渲染、英雄模型/动画、技能图标）
  - 支持鼠标点击移动 + QWER/DF 技能施放
- **英雄与战斗系统**
  - 设计 10 个 MVP 英雄（每个英雄有独立的 Q/W/E/R 技能 + 被动）
  - 实现技能伤害计算、冷却管理（服务器权威）
  - 实现普攻自动寻敌逻辑
  - 实现召唤师技能“闪现”和“雪球”（雪球两段效果）
  - 实现死亡与复活逻辑（8秒固定复活，基地重生）
  - 实现胜利判定（基地摧毁 或 团队击杀达到30）
- **局内社交与 UI**
  - 实现快捷表情轮盘（6个预设文本）
  - 实现实时战绩面板（显示10人 KDA）
  - 实现游戏结束后的结算界面（战绩详情、点赞按钮）
- **积分与数据对接**
  - 调用摸鱼网站现有积分 API（增加/扣除积分）
  - 记录对局结果到数据库（胜/负、KDA、时间）
  - 提供个人数据看板（总场次、胜率、总击杀）
- **断线重连与 AI**
  - 实现玩家断开检测（心跳超时 30 秒）
  - 实现 AI 简单行为树（跟随队友、攻击最近敌人）
  - 实现玩家重连后状态恢复（替换 AI 控制）
- **测试与优化**
  - 5v5 满员压力测试（目标 100 个房间并发）
  - 集成显卡环境下帧率优化（降低粒子特效）
  - 首屏加载优化（分包加载，资源 ≤5MB）
